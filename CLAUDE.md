# xq-parser – developer guide for Claude

## Project overview

This repo generates XQuery parsers for XQuery 3.1 and XQuery 4.0, with optional extensions for eXist-DB and the XQuery Update Facility (XQUF). Parsers are generated by the [REx parser generator](https://github.com/GuntherRademacher/rex-parser-generator) from EBNF grammars.

## Key files

| Path | Purpose |
|------|---------|
| `rawgrammars/XQuery-40.ebnf` | Raw XQuery 4.0 grammar (periodically synced from upstream REx samples) |
| `rawgrammars/XQuery-31.ebnf` | Raw XQuery 3.1 grammar (manually maintained) |
| `transformedgrammars/XQuery-40-full.ebnf` | XQ40 grammar after XQUF/eXist mutations (input to REx) |
| `transformedgrammars/XQuery-31-full.ebnf` | XQ31 grammar after XQUF/eXist mutations (input to REx) |
| `parsers/XQuery-40-full.ts` | Generated XQ40 parser (do not edit by hand) |
| `parsers/XQuery-31-full.ts` | Generated XQ31 parser (do not edit by hand) |
| `src/index.ts` | Transformation pipeline entry point |
| `src/applyMutations.ts` | Mutation engine that splices XQUF/eXist rules into the grammar |
| `src/xq4/xquery4-transformations.ts` | XQ4-specific mutations (Keyword exclusion, XQUF/eXist in XQ4 context) |
| `src/exist-mutations.ts` | eXist-DB mutations for XQ31 |
| `src/xquf-mutations.ts` | XQUF mutations for XQ31 |
| `generate-parsers.sh` | One-shot script to regenerate both parsers |
| `parser_header.txt` | TypeScript header prepended to each generated parser |
| `test/qt4/qt4.test.ts` | W3C QT4 conformance test runner |
| `test/qt4/qt4-snaps/` | Snapshot files recording known parser conformance failures |

## Regenerating the parsers

Any change to `rawgrammars/*.ebnf` or the mutation code under `src/` requires regenerating the parsers with REx.

### 1. Build REx (one-time setup)

REx is not bundled; clone and build it with `g++`:

```sh
git clone https://github.com/GuntherRademacher/rex-parser-generator.git
cd rex-parser-generator
rex/scripts/build-gpp.sh
cd ..
```

The binary will be at `rex-parser-generator/rex/build/bin/rex`.

### 2. Run the generator

```sh
./generate-parsers.sh
```

This script:
1. Transforms each raw grammar through `src/index.ts` (applies XQUF and eXist-DB mutations)
2. Runs `rex -typescript -glalr 1 -a pi -tree` on each transformed grammar
3. Prepends `parser_header.txt` and writes the result to `parsers/`

### 3. Update snapshots if parser behaviour changed

After regenerating, run the W3C conformance tests and update snapshots:

```sh
# Init the submodule if not already done
git submodule update --init qt4tests

QT4_TESTS_DIR=qt4tests npm run update-snapshots
```

Then review the snapshot diff to confirm the changes are expected improvements.

## Grammar update workflow

The XQ40 raw grammar is synced from upstream:
```
https://raw.githubusercontent.com/GuntherRademacher/rex-parser-generator/main/docs/sample-grammars/XQuery-40.ebnf
```

The CI workflow `update-xquery-4-ebnf.yml` (`workflow_dispatch`) automates this:
download latest grammar → apply mutations → run REx → open a PR.

### Adapting mutations for grammar changes

The XQ4 grammar uses `Keyword` (not `ReservedName` as in older versions) to exclude words from `NonKeywordQName`. Mutation `where` values in `src/xq4/xquery4-transformations.ts` must match the grammar's production names:

- Hard keywords → `where: 'Keyword'`
- Function-name augmentation → targets `UnreservedFunctionQName` (still present in new grammar)
- Token delimiters → `where: 'QNameOrKeywordDelimiter'` (still present)

## Running tests

```sh
# Unit + smoke tests only
npm test

# Full W3C QT4 conformance suite (requires submodule)
git submodule update --init qt4tests
QT4_TESTS_DIR=qt4tests npm test

# Regenerate snapshots (after parser changes)
QT4_TESTS_DIR=qt4tests npm run update-snapshots
```

Snapshots in `test/qt4/qt4-snaps/` record known conformance failures (cases where our parser diverges from the W3C spec). A snapshot update PR should include an explanation of which failures were fixed or introduced.
